Maze Generation on the Platonic Solids:
A Three-Layer Graph Model

Kakeami
Georgia Institute of Technology
Technical Report — March 2026 — Source Code

Abstract

We present a systematic framework for generating perfect mazes on the surfaces of all five Platonic solids—the tetrahedron, cube, octahedron, dodecahedron, and icosahedron. The core contribution is a three-layer graph model that separates the problem into (1) a macro face-adjacency graph, (2) per-face grid graphs tailored to each polygon type, and (3) a combined maze graph with a tunable inter-face passage constraint $k$. We show that classical spanning-tree algorithms (Kruskal, DFS backtracker, Wilson) can be applied directly to this unified graph to produce mazes with a guaranteed unique solution. We further introduce a warp tunnel mechanism that adds a shortcut between opposite faces while provably preserving the spanning-tree property, and observe a structural duality between the net-unfolding problem and maze generation—both reduce to finding a spanning tree, albeit on different layers of our model. Finally, we propose polyhedral-specific difficulty metrics (face crossings, face coverage) and provide an open-source interactive demo with PDF net export for paper-craft construction.

1. Introduction

Maze generation on planar grids is a well-studied topic in recreational mathematics and computer science [1, 2]. However, generating mazes on the surfaces of three-dimensional polyhedra introduces challenges that do not arise in the planar case: cells belonging to different faces must be connected through shared geometric edges, the grid topology varies with face shape (squares, triangles, pentagons), and the solver must reason about spatial relationships that are not directly visible from any single viewpoint.

Our work is motivated by a simple hands-on activity: printing a maze on an unfolded polyhedron net, cutting it out, and assembling it into a three-dimensional puzzle. This end-to-end pipeline—from algorithmic generation to physical paper craft—requires solving several interrelated problems: subdividing each face into a grid of cells, connecting grids across face boundaries in a controlled manner, generating a spanning tree over the resulting graph, and unfolding the polyhedron into a printable net.

While cube-surface mazes have been implemented in several hobby and course projects [3, 4, 5], no prior work provides a formal graph-theoretic model that generalizes to arbitrary Platonic solids, controls the number of inter-face passages, or addresses non-rectangular face grids. This paper fills that gap with the following contributions:

  1. A three-layer graph model ($G_{\text{face}}$, $G_f$, $G_{\text{maze}}$) that cleanly separates macro topology from micro grid geometry (Section 3).
  2. Grid schemes for all regular polygon types: RectGrid for squares, TriGrid for equilateral triangles, and PentGrid for regular pentagons (Section 3.2).
  3. A tunable inter-face passage constraint $k$ that controls the number of passages between adjacent faces, acting as a bottleneck parameter (Section 3.3).
  4. A warp tunnel mechanism with a provably tree-preserving splice operation (Section 4.3).
  5. An observation of structural duality between net unfolding and maze generation (Section 5).
  6. Polyhedral difficulty metrics (face crossings, face coverage) that capture aspects unique to surface mazes (Section 6).

2. Related Work

2.1 Planar Maze Generation

The generation of perfect mazes (i.e., spanning trees of a grid graph) is classical. Jamis Buck's comprehensive treatment [1] covers depth-first search (DFS) backtracking, randomized Kruskal, Prim, Aldous–Broder, Wilson's algorithm, and others. Berthier [2] provides a recent survey that extends coverage to topological variants (torus, cylinder, Möbius strip) without, however, proposing a concrete model for polyhedral surfaces.

2.2 Cube-Surface Mazes

Several implementations target the cube specifically. Vijayaraghavan and Krishnamurthy [3] describe a MATLAB-based system that generates a rolling-ball maze on a cube surface using recursive backtracking, with physical fabrication via FDM 3D printing. The carvey_cube_maze project [4] takes a similar approach in Python, producing SVG net outputs for CNC carving. Pullen's Daedalus [5] includes a "planair" mode that wraps a rectangular grid around a cube. All three are limited to cubes and lack formal graph-theoretic modeling or inter-face passage control.

2.3 Mazes on General Surfaces

Li and Mount [6] generate mazes on spherical surfaces using Voronoi tessellations, producing irregular (non-grid) cell layouts. Pedersen and Singh [7] create organic labyrinths on arbitrary 2D manifolds via curve-based growth simulations. Neither approach produces grid-based mazes on polyhedral faces.

PolyJS by Arcus [8] generates mazes on arbitrary polyhedra loaded from OFF files, but treats each face as a single cell—there is no intra-face grid subdivision. This yields mazes with at most as many cells as the polyhedron has faces (e.g., 20 cells for an icosahedron), which is far too coarse for a satisfying puzzle.

2.4 Holonomy Mazes

Segerman's holonomy mazes [9] are direction-dependent puzzles on polyhedra where a token's orientation changes as it crosses edges, gated by the holonomy group. While these are deep mathematical objects on polyhedra, they are fundamentally different from grid-based perfect mazes: there is no spanning tree, no unique solution path, and no grid subdivision.

2.5 Maze Difficulty Metrics

Several authors have proposed quantitative metrics for maze difficulty, though none address the unique challenges of polyhedral surfaces. McClendon [12] introduces a "hallway complexity" measure based on the twistiness of corridors. Buck [1] discusses dead-end count as a structural indicator but does not measure how deep dead-end branches extend—i.e., how far a solver must travel before discovering a wrong turn. Pullen [5] informally mentions a "deviation distance" concept but provides no formal definition. Kim and Crawfis [11] propose an evaluation framework for maze generation that considers solution path properties but does not account for branch depth. To our knowledge, no prior work formalizes the depth of dead-end branches as a difficulty metric—a gap we revisit in our discussion of future work (Section 8).

2.6 Patents

U.S. Patent 4,008,895 [10] (1977) describes a physical puzzle with passageways on polyhedral faces connected across shared edges for a rolling-ball maze. The patent covers the physical artifact but does not address automated generation, grid subdivision, or algorithmic methods.

3. Three-Layer Graph Model

We decompose the maze-generation problem on a Platonic solid $P$ into three layers. Each layer addresses a distinct aspect of the problem, and together they yield a single graph on which standard spanning-tree algorithms can operate.

3.1 Layer 1: Face Adjacency Graph (Macro)

Definition 1 (Face Adjacency Graph). Let $P = (V_P, E_P, F_P)$ be a Platonic solid with vertex set $V_P$, edge set $E_P$, and face set $F_P$. The face adjacency graph is $G_{\text{face}} = (F_P, A)$ where $(f_i, f_j) \in A$ if and only if faces $f_i$ and $f_j$ share an edge in $E_P$.

This is precisely the dual graph of $P$. Each node represents a face, and each edge represents a shared geometric boundary. The face adjacency graph captures the macro-level topology of the polyhedron.

Platonic Solid $|F_P|$ $|A|$ Face Shape Opposite Pairs
Tetrahedron46Triangle0

Two faces $f_i, f_j$ are opposite if their outward normals $\mathbf{n}_i, \mathbf{n}_j$ satisfy $\mathbf{n}_i \cdot \mathbf{n}_j < -1 + \varepsilon$ for a small tolerance $\varepsilon$. The tetrahedron has no opposite faces; all other Platonic solids do. Opposite faces play a key role in the warp mechanism (Section 4.3).

3.2 Layer 2: Per-Face Grids (Micro)

Definition 2 (Face Grid). For each face $f \in F_P$, a face grid $G_f = (C_f, W_f)$ is a graph whose nodes $C_f$ are cells and whose edges $W_f$ connect adjacent cells within the face. Each cell is identified by a tuple $(f, r, c)$ where $r$ is the row index and $c$ is the column index in a face-local coordinate system.

The grid type is determined by the face shape. We define three grid schemes, one for each polygon type that appears among the Platonic solids.

Novel contribution While rectangular grids on square faces are straightforward, the TriGrid and PentGrid schemes for triangular and pentagonal faces are, to our knowledge, the first proposed for polyhedral maze generation.

3.2.1 RectGrid (Square Faces — Cube)

A square face is subdivided into an $n \times n$ orthogonal grid, yielding $n^2$ cells per face. Cell $(f, r, c)$ is adjacent to its four cardinal neighbors. The cell center in 3D is computed via bilinear interpolation:

$$ \mathbf{p}(r, c) = \mathbf{v}_0 + \frac{r + 0.5}{n}\,(\mathbf{v}_3 - \mathbf{v}_0) + \frac{c + 0.5}{n}\,(\mathbf{v}_1 - \mathbf{v}_0) $$

where $\mathbf{v}_0, \mathbf{v}_1, \mathbf{v}_3$ are face vertices defining the local frame.

3.2.2 TriGrid (Triangular Faces — Tetrahedron, Octahedron, Icosahedron)

An equilateral triangular face is subdivided into $n^2$ smaller triangles arranged in $n$ rows. Row $r$ ($0 \le r < n$) contains $2r + 1$ cells: $r + 1$ upward-pointing triangles ($\triangle$, even column index) and $r$ downward-pointing triangles ($\nabla$, odd column index).

The adjacency structure is:

  • Within a row: consecutive cells $(r, c)$ and $(r, c+1)$ share an edge.
  • Between rows: an upward triangle $\triangle(r, 2k)$ shares its base with the downward triangle $\nabla(r+1, 2k+1)$ in the row below.

Cell centers are computed via barycentric coordinates within the face triangle $(\mathbf{v}_0, \mathbf{v}_1, \mathbf{v}_2)$:

$$ \mathbf{p}(r, c) = \alpha\,\mathbf{v}_0 + \beta\,\mathbf{v}_1 + (1 - \alpha - \beta)\,\mathbf{v}_2 $$

where $(\alpha, \beta)$ are obtained by solving the least-squares projection $\alpha\,\mathbf{u} + \beta\,\mathbf{v} \approx \mathbf{p}$ with $\mathbf{u} = \mathbf{v}_1 - \mathbf{v}_0$ and $\mathbf{v} = \mathbf{v}_2 - \mathbf{v}_0$.

3.2.3 PentGrid (Pentagonal Faces — Dodecahedron)

A regular pentagonal face has no natural orthogonal subdivision. We decompose it into five isosceles triangular sectors by connecting the face center $\mathbf{c}_f$ to each vertex pair $(\mathbf{v}_s, \mathbf{v}_{s+1})$. Each sector is then subdivided as a TriGrid with resolution $n$, yielding $5n^2$ cells per pentagonal face.

The sectors are stitched together along their shared radial edges: the rightmost column of sector $s$ is connected to the leftmost column of sector $(s+1) \bmod 5$. Additionally, the five apex cells (row 0 of each sector) form a central ring.

3.3 Layer 3: Combined Maze Graph

Definition 3 (Combined Maze Graph). The combined maze graph is $$ G_{\text{maze}} = \Bigl(\;\bigcup_{f \in F_P} C_f,\;\; E_{\text{intra}} \cup E_{\text{inter}}\Bigr) $$ where $E_{\text{intra}} = \bigcup_{f} W_f$ collects all intra-face edges, and $E_{\text{inter}} = \bigcup_{(f_i, f_j) \in A} B_{ij}$ collects inter-face edges with the constraint $|B_{ij}| \le k$ for each adjacent pair.
Novel contribution The inter-face passage constraint $k$ is a key parameter with no counterpart in prior maze generators. By varying $k$, the puzzle designer controls a trade-off between two aspects of difficulty. Small $k$ creates bottlenecks that force long detours within each face, increasing solution path length. Larger $k$ values admit more inter-face passages, increasing face crossings (Section 6) and the associated cognitive load of spatial reorientation—the solver must repeatedly re-orient themselves as they move between faces whose spatial relationships are not directly visible.

3.3.1 Boundary Cell Mapping

For two adjacent faces $f_i$ and $f_j$ sharing a geometric edge $e = (\mathbf{v}_a, \mathbf{v}_b)$, we must identify which cells on each face lie along this shared boundary. The mapping proceeds as follows:

  1. Project $\mathbf{v}_a$ and $\mathbf{v}_b$ into the local coordinate frame of $f_i$ to determine which row/column of the grid lies along $e$.
  2. Enumerate the $n$ boundary cells of $f_i$ along $e$, ordered from $\mathbf{v}_a$ toward $\mathbf{v}_b$.
  3. Repeat for $f_j$. The two ordered lists of boundary cells are then aligned pairwise.

For RectGrid, boundary detection uses the projected coordinates to identify which of the four sides (top, bottom, left, right) the edge falls on. For TriGrid, the projection via least-squares determines which of the three triangle edges ($\mathbf{v}_0$–$\mathbf{v}_1$, $\mathbf{v}_0$–$\mathbf{v}_2$, or $\mathbf{v}_1$–$\mathbf{v}_2$) is involved. For PentGrid, each sector's bottom edge is tested against the 3D edge.

3.3.2 Passage Selection ($k$-Constraint)

Given $n$ aligned boundary cell pairs for an adjacent face pair, we select $k$ evenly spaced pairs as inter-face edges $B_{ij}$:

  • If $k \ge n$: all $n$ pairs are selected.
  • If $k = 1$: the middle pair $\lfloor n/2 \rfloor$ is selected.
  • Otherwise: pairs at indices $\bigl\lfloor i \cdot (n-1) / (k-1) \bigr\rceil$ for $i = 0, 1, \ldots, k-1$.

3.3.3 Maze as Spanning Tree

Definition 4 (Perfect Maze). A perfect maze on $G_{\text{maze}}$ is a spanning tree $T$ of $G_{\text{maze}}$. As a tree, $T$ is connected, acyclic, and has exactly $|V| - 1$ edges. Every pair of cells is connected by a unique path—ensuring a unique solution.

This formulation means that any spanning-tree algorithm can generate a maze on $G_{\text{maze}}$, regardless of the underlying polyhedron or grid type. The three-layer separation ensures that the algorithm need not be aware of the geometric details; it operates on a single undirected graph.

4. Maze Generation

4.1 Spanning-Tree Algorithms

We employ three classical algorithms, each producing a spanning tree of $G_{\text{maze}}$ with distinct structural characteristics.

4.1.1 Randomized Kruskal

All edges of $G_{\text{maze}}$ are shuffled randomly. A Union-Find structure is initialized with each cell as a singleton. Edges are processed in order; an edge $(u, v)$ is added to $T$ if $u$ and $v$ belong to different components. The algorithm terminates when $T$ has $|V| - 1$ edges.

Properties: Many short dead ends, relatively uniform branch distribution. Time complexity $O(|E| \log |E|)$.

4.1.2 DFS Backtracker

A stack-based depth-first traversal starts from a random cell. At each step, the current cell picks a random unvisited neighbor and carves a passage; if no unvisited neighbor exists, the algorithm backtracks. This continues until all cells are visited.

Properties: Long winding corridors, fewer dead ends. Time complexity $O(|V|)$. Generates mazes with the longest average solution paths among the three algorithms.

4.1.3 Wilson's Algorithm (Loop-Erased Random Walk)

Wilson's algorithm produces a uniform random spanning tree—each spanning tree of $G_{\text{maze}}$ is equally likely. Starting from a single-cell tree, it repeatedly performs random walks from unvisited cells until hitting the tree, erasing any loops formed during the walk, and adding the resulting path to the tree.

Properties: Unbiased distribution over all spanning trees. Expected time $O(|V|)$ but with high variance; worst case $O(|V|^3)$.

4.2 Start and Goal Placement

To maximize path length and encourage exploration of the entire polyhedron, we use a double-BFS diameter approximation:

Algorithm 1: Start/Goal Placement
  1. Pick an arbitrary cell $v_0$. Run BFS on $T$ from $v_0$; let $v_1$ be the farthest cell.
  2. Run BFS on $T$ from $v_1$; let $v_2$ be the farthest cell on a different face than $v_1$.
  3. Set $\text{start} = v_1$, $\text{goal} = v_2$.

The different-face constraint ensures that the solver must cross at least one face boundary, leveraging the three-dimensional structure of the puzzle.

4.3 Warp Tunnel

For polyhedra with opposite face pairs, we introduce an optional warp tunnel: a single bidirectional passage connecting cells on opposite faces. Physically, this corresponds to piercing aligned holes through the assembled polyhedron and threading a straw between them.

Novel contribution The warp mechanism is unique to polyhedral mazes and has no planar analogue. The key challenge is adding this edge to the spanning tree $T$ without creating a cycle (which would destroy the unique-solution property).

4.3.1 Opposite Cell Computation

Given a cell on face $f_i$ at position $\mathbf{p}$, the corresponding cell on the opposite face $f_j$ is found by perpendicular projection through the polyhedron:

$$ \mathbf{p}' = \mathbf{p} - 2\,(\mathbf{p} - \mathbf{c}) \cdot \mathbf{n}_i \;\mathbf{n}_i $$

where $\mathbf{c}$ is the polyhedron center and $\mathbf{n}_i$ is the outward normal of $f_i$. The target cell on $f_j$ is the one whose 3D center is nearest to $\mathbf{p}'$.

4.3.2 Candidate Selection (Tiered Strategy)

Not all cell pairs make good warp endpoints. We use a three-tier prioritization:

TierCandidate PoolConstraint
1 (best)Interior dead-end cellsOpposite cell is also an interior dead end
2Interior cellsOpposite cell is also interior
3 (fallback)All cellsNone

Within each tier, we select the pair $(c_A, c_B)$ that maximizes the spanning-tree distance $d_T(c_A, c_B)$, ensuring the warp provides the greatest shortcut. The tiered search guarantees 100% placement success across all polyhedra and seeds.

4.3.3 Tree-Preserving Splice

Proposition 1 (Splice Preserves Spanning Tree). Let $T$ be a spanning tree of $G_{\text{maze}}$ and let $(c_A, c_B)$ be two cells connected by a unique path $\pi = (c_A, u_1, \ldots, u_m, c_B)$ in $T$. Define $\text{mid} = \lfloor m/2 \rfloor$. Then $$ T' = T - \{(u_{\text{mid}}, u_{\text{mid}+1})\} + \{(c_A, c_B)\} $$ is also a spanning tree of $G_{\text{maze}} + \{(c_A, c_B)\}$.

Proof sketch. Removing any edge from a tree disconnects it into exactly two components. The path $\pi$ in $T$ connects $c_A$ to $c_B$; removing the edge at its midpoint splits $T$ into two subtrees, one containing $c_A$ and the other $c_B$. Adding the warp edge $(c_A, c_B)$ reconnects the two subtrees. The result has $|V| - 1$ edges and is connected, hence it is a spanning tree. $\square$

The midpoint choice for edge removal balances the sizes of the two subtrees, minimizing disruption to the overall maze structure.

5. Unfolding Duality

Novel contribution We observe a structural duality between the net-unfolding problem and maze generation: both reduce to finding spanning trees, but on different layers of the graph model.

An unfolding (or net) of a polyhedron $P$ is a planar layout of all faces obtained by cutting along a subset of edges and flattening. A valid net must be connected and non-overlapping. The combinatorial structure of a net is determined by which edges are cut and which are folded:

  • Fold edges: edges where adjacent faces remain attached. These form a spanning tree of $G_{\text{face}}$.
  • Cut edges: edges where the surface is separated. These are the complement of the spanning tree.

Thus, choosing a net is equivalent to choosing a spanning tree of the Layer-1 face adjacency graph $G_{\text{face}}$. Meanwhile, generating a maze is choosing a spanning tree of the Layer-3 combined graph $G_{\text{maze}}$. The two spanning trees operate on different layers of the same model but serve dual purposes:

Net UnfoldingMaze Generation
Graph$G_{\text{face}}$ (Layer 1)$G_{\text{maze}}$ (Layer 3)
Tree edgesFold lines (faces stay attached)Passages (cells connected)
Non-tree edgesCut lines (glue tabs)Walls (cells separated)
ResultFlat, connected netPerfect maze with unique path

In our implementation, we exploit this duality: the same BFS-based tree traversal used to lay out the net (reflecting child faces across shared edges) can also inform the rendering pipeline by identifying which face boundaries are "cut" and thus require glue tabs in the printed output.

6. Difficulty Metrics

Novel contribution Traditional maze metrics (solution length, dead-end count) do not capture the three-dimensional navigation challenge. We introduce polyhedral-specific metrics.

Given a maze $T$ on polyhedron $P$ with start cell $s$ and goal cell $g$, let $\pi(s, g)$ denote the unique $s$–$g$ path in $T$. We define:

Metric Definition Interpretation
Solution length $|\pi(s,g)|$ Number of cells in the solution path
Solution ratio $|\pi(s,g)| \;/\; |V|$ Fraction of cells visited by the solution
Dead-end count $|\{v \in V : \deg_T(v) = 1\}|$ Cells with only one passage (traps)
Dead-end density Dead-end count $/ \; |V|$ Fraction of cells that are dead ends
Face crossings $|\{i : \text{face}(\pi_i) \ne \text{face}(\pi_{i+1})\}|$ How often the solution crosses face boundaries
Face coverage $|\{\text{face}(v) : v \in \pi(s,g)\}| \;/\; |F_P|$ Fraction of faces visited by the solution
Warp used $\mathbf{1}[(c_A, c_B) \in \pi(s,g)]$ Whether the solution passes through the warp tunnel

The face crossings and face coverage metrics are unique to polyhedral surface mazes. High face crossings indicate that the solver must repeatedly re-orient themselves as they move between faces—a cognitive challenge absent in planar mazes. High face coverage ensures the puzzle makes use of the entire polyhedron rather than being confined to a subset of faces.

7. Implementation

7.1 System Overview

The system is implemented in TypeScript with a clean separation between a DOM-independent core (core/) containing all graph and geometric algorithms, and rendering modules (render/, ui/) that handle Three.js visualization and browser interaction.

7.2 Determinism

All randomized operations use a seeded PRNG (Mulberry32), ensuring full reproducibility: the same seed and parameters always produce the identical maze. This is essential for sharing puzzles via URL-encoded parameters and for enabling systematic comparisons.

7.3 Net Export and Paper Craft

The system generates a two-page A4 landscape PDF: page 1 shows the maze net without the solution (for solving), and page 2 includes the solution path and warp markers (for checking). The net includes trapezoidal glue tabs on cut edges, computed from the Layer-1 spanning tree (unfolding duality). The tab geometry uses a 15% height ratio with 50% taper to prevent overhang during folding.

7.4 Interactive Demo

An interactive web demo is available at kakeami.github.io/polyhedral-maze. Users can select the polyhedron type, grid resolution $n$, passage count $k$, generation algorithm, and random seed. The 3D visualization uses Three.js with OrbitControls, and mazes can be exported as PDF nets for physical construction.

8. Conclusion and Future Work

We have presented a three-layer graph model for generating perfect mazes on the surfaces of all five Platonic solids. The model cleanly separates the macro topology (face adjacency), micro geometry (per-face grids), and combined maze structure, enabling any spanning-tree algorithm to produce mazes regardless of the underlying polyhedron. The inter-face passage constraint $k$ provides a principled control over maze difficulty, and the warp tunnel mechanism adds a novel gameplay element while provably preserving the unique-solution guarantee.

Several directions remain open for future investigation:

  • Archimedean solids: The three-layer model naturally extends to semi-regular polyhedra with mixed face types (e.g., the truncated icosahedron with hexagonal and pentagonal faces), requiring only new grid implementations for additional polygon types.
  • Difficulty analysis: A systematic experimental comparison of algorithms (Kruskal, DFS, Wilson) across polyhedra and parameter settings ($n$, $k$), using the metrics defined in Section 6, would quantify how maze structure varies with these factors.
  • User studies: The physical paper-craft aspect invites studies of how three-dimensional maze solving affects spatial reasoning, particularly in educational contexts with children.
  • Cognitive load metric: A formal metric capturing the mental effort of tracking connectivity across hidden faces of the assembled polyhedron would complement the current face-crossing count.
  • Dead-end branch depth: A formal metric capturing the depth of dead-end branches—how far a solver must travel before discovering a wrong turn—would complement the current face-crossing count and could guide the design of difficulty-optimized generation algorithms. As noted in Section 2, no prior work formalizes this notion, suggesting an opportunity for both new metrics and new algorithms that maximize "meanness."
